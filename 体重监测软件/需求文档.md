# 体重监测软件 — 需求文档

## 1. 项目概述

### 1.1 项目名称
**轻重** — 个人体重监测微信小程序

### 1.2 项目定位
一款**全自动**的个人体重追踪工具。绑定小米/米家 WiFi 智能秤后，每次上秤数据自动同步——无需手动输入，打开小程序就能看到最新体重和趋势变化。

### 1.3 核心差异点
> **零操作**：上秤 → 秤自动上传数据到小米云 → 小程序自动拉取 → 你看到结果。
> 整个过程只有"站上去"这一个动作。

### 1.4 目标用户
- 使用小米/米家智能秤的用户
- 关注体重变化的健身/减脂人群
- 家庭成员共用一台秤（V2 支持多成员识别）

### 1.5 技术方案

```
┌──────────┐    WiFi     ┌──────────────┐    API轮询    ┌────────────────┐
│ 小米智能秤 │ ─────────→ │  小米云服务    │ ←────────── │ 云函数(定时任务) │
└──────────┘             └──────────────┘              └───────┬────────┘
                                                              │ 写入
                                                     ┌────────▼────────┐
                                                     │ 微信云数据库     │
                                                     └────────┬────────┘
                                                              │ 查询
                                                     ┌────────▼────────┐
                                                     │ 微信小程序       │
                                                     │ (本地Storage缓存) │
                                                     └─────────────────┘
```

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | 微信小程序原生 + TypeScript | 性能最优 |
| 本地存储 | wx.Storage API | 离线可看历史数据 |
| 云端数据库 | 微信云开发 Cloud Base | 免运维、自带鉴权 |
| 数据桥梁 | 云函数 + 小米云 API | 定时从小米云拉取体重数据 |
| 小米账号 | 小米 OAuth 2.0 | 用户授权后获取设备数据 |
| 图表组件 | ECharts 小程序版 | 趋势折线图、月度统计 |
| UI 组件 | 自定义 | 简洁轻量 |

### 1.6 小米秤接入方案

小米 WiFi 智能秤（如小米体脂秤 2、小米八电极体脂秤等）的工作流程：

1. **秤端**：用户上秤测量 → 数据通过 WiFi 自动上传到小米云
2. **授权**：用户在小程序中登录小米账号（OAuth），授权读取设备数据
3. **同步**：云函数使用用户授权 Token，定期从小米云 API 拉取最新体重数据
4. **存储**：拉取到的数据存入微信云数据库，同时缓存到小程序本地 Storage

**支持的数据项**（取决于秤型号）：
- 体重（kg）—— 所有型号
- BMI —— 所有型号
- 体脂率 —— 体脂秤型号
- 肌肉量、骨量、水分率、基础代谢、内脏脂肪等级等 —— 高端型号

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 设备绑定
- 引导用户登录小米账号（OAuth 授权页面）
- 授权后列出账号下的智能秤设备
- 选择要绑定的设备，确认绑定
- 绑定后自动触发首次数据同步
- 支持解绑和更换设备

#### 2.1.2 自动数据同步
- **同步方式**：云函数定时任务（建议每小时一次）+ 小程序启动时主动拉取
- **同步内容**：从上一次同步时间至今的所有新记录
- **后台静默**：用户无需任何操作，打开即看最新数据
- **同步状态**：首页显示"最后同步时间"，手动下拉可强制刷新
- **智能去重**：同一天多次上秤取最新一条（或全部保留，取决于用户偏好）

#### 2.1.3 体重趋势图
- **主图表**：以日期为横轴、体重为纵轴的折线图
- **时间范围切换**：7 天 / 30 天 / 90 天 / 半年 / 全年 / 全部
- **交互**：点击数据点显示当日体重值
- **辅助线**：目标体重虚线标注
- **数据摘要**：所选时间范围内的最高/最低/平均值

#### 2.1.4 历史记录
- **列表展示**：按日期倒序排列，每条显示日期、体重、变化量（相比前一天）
- **筛选**：按月筛选
- **删除**：左滑删除（仅删除小程序侧数据，不影响小米云）
- **备注**：支持为某条记录添加文字备注

#### 2.1.5 目标管理
- **目标体重**：设定一个目标值（kg）
- **目标达成进度**：百分比环图或进度条展示
- **预计达成**：基于近期变化速率估算预计达成日期（仅供参考）

---

### 2.2 辅助功能

#### 2.2.1 数据看板（首页）
- 今日体重（大号字体醒目展示）
- 相比昨日的升降（绿色降/红色升，带箭头）
- 最后同步时间 + 同步状态
- 7 日变化量
- 历史最低 / 历史最高
- BMI 指数（从秤数据获取或自行计算）
- 体脂率（如有）

#### 2.2.2 身体数据详情
- 如果秤支持，展示多维度身体数据卡片：
  - 体重、BMI、体脂率、肌肉量、水分率
  - 骨量、基础代谢、内脏脂肪等级、蛋白质率
- 每项数据可展开查看历史趋势

#### 2.2.3 个人信息
- 昵称、头像（微信授权获取）
- 身高（体脂秤计算 BMI 需要身高，从秤数据获取或手动填写）
- 性别、年龄（可选）

#### 2.2.4 数据提醒
- **未称重提醒**：如果当天还未检测到体重数据，微信服务通知提醒
- **目标达成提醒**：达到阶段目标时推送通知

---

### 2.3 扩展功能（V2）

#### 2.3.1 手动补录
- 当秤数据缺失或使用其他秤时，可手动录入体重
- 手动记录标记为"手动"以区分自动同步数据

#### 2.3.2 数据导出
- 导出为 CSV 文件，可保存到微信文件或发送到邮箱

#### 2.3.3 家庭多成员
- 小米秤部分型号支持自动识别不同家庭成员（根据体重判断）
- 小程序端支持切换成员，查看各自数据

#### 2.3.4 照片记录
- 对标"身材变化"，支持拍照记录正面/侧面照片
- 按日期排列，可滑动对比不同时期身材变化

## 3. 数据模型

### 3.1 体重记录（weight_records）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| _id | string | ✓ | 云数据库自动生成 |
| openid | string | ✓ | 微信用户标识 |
| weight | float | ✓ | 体重（kg），精确到 0.1 |
| bmi | float | | BMI 指数 |
| body_fat | float | | 体脂率（%） |
| muscle_mass | float | | 肌肉量（kg） |
| water_rate | float | | 水分率（%） |
| bone_mass | float | | 骨量（kg） |
| bmr | int | | 基础代谢（kcal） |
| visceral_fat | int | | 内脏脂肪等级 |
| protein_rate | float | | 蛋白质率（%） |
| measure_date | string | ✓ | 测量日期 YYYY-MM-DD |
| measure_time | string | | 测量时间 HH:mm:ss |
| source | string | ✓ | 数据来源：`xiaomi_cloud` / `manual` |
| xiaomi_id | string | | 小米云侧的数据 ID（去重用） |
| note | string | | 用户备注（最多 100 字） |
| created_at | datetime | ✓ | 入库时间 |
| updated_at | datetime | | 更新时间 |

### 3.2 用户信息（user_profile）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| openid | string | ✓ | 微信用户标识 |
| nickname | string | | 昵称 |
| avatar_url | string | | 头像 URL |
| height | float | | 身高（cm），可从秤获取 |
| gender | string | | male / female |
| birthday | string | | YYYY-MM-DD |
| target_weight | float | | 目标体重（kg） |
| reminder_enabled | boolean | | 是否开启未称重提醒 |

### 3.3 小米账号绑定（xiaomi_binding）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| openid | string | ✓ | 微信用户标识 |
| xiaomi_user_id | string | ✓ | 小米账号用户 ID |
| access_token | string | ✓ | 小米 OAuth access_token（加密存储） |
| refresh_token | string | ✓ | 小米 OAuth refresh_token（加密存储） |
| token_expires_at | datetime | ✓ | Token 过期时间 |
| device_id | string | ✓ | 绑定的秤设备 ID |
| device_name | string | | 设备名称（如"小米体脂秤2"） |
| last_sync_at | datetime | | 最后一次同步时间 |
| bind_status | string | ✓ | active / expired / revoked |
| created_at | datetime | ✓ | 绑定时间 |

### 3.4 本地存储策略
- 体重历史数据全量缓存到本地 Storage
- 打开小程序先展示本地数据，同时后台查云端增量
- 云端数据覆盖本地（云端为准）
- 小米 Token 仅存储在云数据库，小程序本地不保存敏感凭证

## 5. 交互与视觉

### 5.1 设计风格
- **关键词**：简洁、轻量、自动感
- **色调**：柔和绿色主色（健康感），浅灰背景
- **字体**：系统默认中文字体，数字使用等宽字体
- **图表**：低饱和度配色，线条流畅

### 5.2 交互原则
- **零操作优先**：打开即看，不需要任何点击就能看到最新数据
- 首页下拉刷新触发手动同步（有加载动画）
- 体重升降：绿色箭头↓表示下降，红色箭头↑表示上升
- 首次使用流程：微信授权 → 绑定小米账号 → 选择设备 → 自动同步 → 进入首页
- 删除操作需二次确认

## 6. 小米云 API 对接要点

### 6.1 认证方式
- 使用小米 OAuth 2.0 标准授权流程
- 云函数负责 Token 管理和刷新
- Token 加密存储在微信云数据库

### 6.2 数据获取
- 调用小米开放平台设备数据 API
- 按时间范围查询体重历史记录
- 云函数定时触发（微信云开发定时触发器，建议每小时）
- 同时支持小程序主动调用云函数立即同步

### 6.3 实测注意事项
- 小米 API 频率限制需实测确认
- 需在小米开放平台注册应用（[https://developers.xiaoai.mi.com](https://developers.xiaoai.mi.com)）
- 部分 API 可能需要企业认证，需提前确认
- 备选方案：若官方 API 受限，可考虑通过小米账号模拟登录方式获取数据（非官方路径）

## 8. 风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| 小米 API 不对个人开发者开放 | 无法自动获取数据 | 降级为手动输入 + 照片 OCR 识别体重 |
| 小米 OAuth Token 频繁过期 | 同步中断 | 云函数自动刷新 Token + 过期告警 |
| API 调用频率限制 | 同步延迟 | 合理设置轮询间隔 + 启动时主动拉取兜底 |
| 秤未连接 WiFi | 数据缺失 | 当日未检测到体重时，微信提醒用户检查 |
